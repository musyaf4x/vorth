# Vorth Stack Routing

Vorth is the repository activation, memory, and routing layer. It decides which stack should participate; it does not replace the stacks.

## Stack Map

- Superpowers: baseline software engineering process.
- CodeGraph: codebase intelligence before broad exploration.
- ECC: engineering specialist review and risk gates.
- Impeccable: frontend/UI design execution and quality gate.
- Layers: product/UX decision-discovery gate.
- Agy Native Bridge: Antigravity-only bounded execution worker.

## Routing Rules

- Small clear engineering task: use Superpowers minimally; skip CodeGraph, Layers, Impeccable, and ECC unless risk appears.
- Unknown codebase area: use CodeGraph before broad file reads.
- Architecture, test strategy, security, build failure, or cross-module risk: use the relevant ECC specialist gate.
- Frontend/UI work: use Impeccable for shaping, critique, audit, polish, harden, layout, accessibility, responsive behavior, empty states, and design-system fit.
- Product/UX ambiguity: use Layers before Impeccable or implementation.
- Bounded execution in Antigravity: use Agy Native Bridge only after scope, files, acceptance criteria, and verification are clear.

## Precedence

- The user's explicit instruction wins.
- Superpowers owns process.
- CodeGraph decides where to look first when code scope is unclear.
- Layers clarifies product/UX decisions before UI/code when needed.
- Impeccable improves UI/frontend quality after product direction is clear.
- ECC reviews engineering risk.
- Agy Native Bridge executes; it does not decide.
