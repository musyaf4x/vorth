# Vorth Caveman Policy

Caveman is Vorth's compact-report mode. It compresses communication style; it must not reduce reasoning quality.

## When To Use

Use Caveman only for:

- Subagent summaries.
- Handoff notes.
- Short status updates.
- Compact internal reports where ambiguity is low.

## When To Avoid

Do not use Caveman for:

- Main user-facing analysis unless the user asks for terse output.
- Architecture or product reasoning.
- Security, privacy, data-loss, migration, or irreversible-action warnings.
- Multi-step instructions where compact wording could cause mistakes.

## Auto-Clarity

When clarity and brevity conflict, clarity wins.

## Memory Compression

Caveman memory compression is explicit-only. Do not compress README, AGENTS.md, GEMINI.md, Vorth instructions, or project docs unless the user asks for that exact operation.
